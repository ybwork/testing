1. Тестирование

	Тестирование - это последовательность действий нужных для получения информации о качестве продукта и на поиск ошибок в нем.

	Продуктом может быть сайт, мобильное приложение, desktop приложение и т.д.

	По сути любой вещь, которую можно протестировать может считаться продуктом. Например самолет или конструктор лего.

2. Use case

	Юзкейс (Use Case) - это вариант использования системы (сайта, приложения) для достижения конкретной цели.

	Пример

		Цель: удалить 3 продукта из списка.

		Use case:

			Администратор системы заходит на страницу "продукты".

			Администратор системы видит список продуктов.

			Администратор системы выбирает 3 любых продукта.

			Администратор системы нажимает кнопку удалить.

			Удаленные продукты исчезают из списка.

	При составлении use case необходимо часто задавать себе вопрос "А что будет если?"

		Например, а что будет если я удалю все продукты? или а что будет если пользователь открыл страницу продукта, я в этот момент удалил продукт, а после этого пользователь нажал кнопку купить?

		Вопрос "А что будет если?" помогает составить, как можно больше use cases и находит места в системе, которые могут сломаться или работать неправильно.

4. Алгоритм работы

	Заказчик или программист ставит задачу, например проверить работу фильтров.

	Узнаем у постановщика задачи, как это должно работать

	Изучаем то, что нужно проверить и составляем все возможные use cases.

	Проверяем каждый use case.

	Если нет ошибок, то функционал считается протестированным.

	Если есть ошибки, то пишем bug report.
